# Idea → MVP

AI 驱动的产品全流程 Skill，帮助你从发现灵感到验证想法再到实现最小可行产品，完成从 Idea 到 MVP 的完整旅程。

## 安装

```bash
# 安装
npx skills add MorvanZhou/idea2mcp

# 更新
npx skills update
```

## 三阶段工作流

### 阶段一：灵感发现（Find Ideas）

通过多平台搜索（Product Hunt、GitHub Trending、小红书、V2EX、即刻、少数派、Indie Hackers 等）发现近期流行的实用小工具和独立产品，分析痛点和模式，生成可拓展的产品 Ideas。

沟通完成后可生成**灵感确认文档**，记录用户选定的方向、产品偏好、关键问题，作为后续阶段的上下文。

### 阶段二：想法验证（Validate Ideas）

以交互式咨询的方式，通过七步流程验证想法可行性：

1. 想法澄清 → 2. 市场调研 → 3. 需求验证 → 4. 技术可行性 → 5. 商业模式 → 6. 风险评估 → 7. 综合评分

每步都与用户确认，发现障碍时不做简单否定，而是触发 Idea 扩展机制推荐替代方向。

### 阶段三：实现 MVP（Build MVP）

基于验证结果，用用户能理解和维护的技术方案实现最小可行产品。优先选择用户熟悉的技术栈，提供能独立操作的启动和部署指引。涉及前端界面时遵循专属设计规范。

## 核心特性

- **灵活切入**：没想法从阶段一开始，有想法从阶段二，想动手从阶段三
- **用户画像持续维护**：全流程自然捕捉用户的技术背景、行业经验、认知水平，适配沟通深度
- **上下文不丢失**：灵感确认文档 + 用户画像文件确保跨阶段信息延续
- **实时调研**：主动搜索竞品、市场数据和社区讨论
- **适配用户技术水平**：阶段三根据用户背景选择技术栈和交付形式
